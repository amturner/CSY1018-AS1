HTTP

The HyperText Transfer Protocol (HTTP) is the protocol that essentially forms the backbone of the web. Based on the client-server model, and considered to be an application layer (Layer 7) protocol in the OSI Model, HTTP allows for documents that make up web pages, such as HTML and CSS documents, to be transmitted by a server, commonly known as the Web Server, to the client that initially made the HTTP request. Once a client has received the documents that they requested, the result is displayed in the client's web browser.

When hosting files, a Web Server typically uses HTTP on TCP Port 80. However, HTTP Secure (HTTPS), which is an encrypted version of the standard HTTP protocol, instead uses TCP Port 443.

Any communication done over HTTPS is usually encrypted using Secure Sockets Layer (SSL) or the more recent Transport Layer Security (TLS), and is therefore the most widely recommended option when it comes to setting up a web server, especially if sensitive information was to be transferred between the server and client.

WWW

The World Wide Web (WWW) is the term that is commonly used to refer to the many HTML resources, much of which are interlinked, that can be accessed via the Internet

Whilst some people may use this term interchangeably with "Internet", the terms Internet and WWW do in fact have different meanings. Whilst WWW refers to interlinked resources, often coming in the form of HTML web pages, the term Internet refers to the infrastructure that allows user to access the WWW.

Techopedia

Internet

The Internet is the term that is commonly used to refer to the network of globally interconnected devices that allows for data to be exchanged regardless of the device's physical location. Any device that is connected to the Internet transmits data using the Transmission Control Protocol/Internet Protocol (TCP/IP) protocol suite.

Techopedia

HTML

HyperText Markup Language (HTML) is a markup language that is used to create the structure of many of the websites you see on the web today.

For the basic structure of a website, HTML makes use of elements such as <html>, <head> and <body>. When describing content on a page, however, HTML also makes use of semantic elements such as <header>, <nav>, <main> and <footer>

As an example of how HTML code is typically structured, the following code shows you how to make a basic web page with the text "Hello World!":

<!DOCTYPE html>
<html>
    <head>
    </head>
    <body>
        <p>Hello World!</p>
    </body>
</html>

With these elements, HTML can therefore essentially be thought of as the skeleton of a web page.

CSS

Cascading Style Sheets (CSS) is a stylesheet language. Whilst HTML can be thought of as the skeleton of a web page, CSS can essentially be thought of as the skin of a web page.

Unlike HTML code, which makes use of elements like <header> and <footer>, CSS code is instead made up of selectors, which in turn contain properties.

As an example of how a typical CSS declartion is structured, the follow snippet of code shows you how you would make the background of a paragraph black, and the text white:

p {
    background-color: black;
    color: white;
}

JavaScript

JavaScript is a programming language that is commonly used by Web Developers to provide additional functionality on a web page.

Unlike HTML and CSS, which are a markup language and a stylesheet language respectively, the fact that JavaScript is a programming language means that JavaScript code can be executed outside of the user's web browser. As a result, this can give developers the ability to create standalone JavaScript applications.

Stylesheet

A Stylesheet is a document that is made up of CSS code. Stylesheets usually come in the form of files that end with the .css extension.

To add a Stylesheet to an HTML document, you should add a <link> element that references the Stylesheet that you wish to use between the <head> tags, as shown below:

<link href="style.css" type="text/css" rel="stylesheet">

Whilst the example shown here shows the Stylesheet "style.css" being imported from the website's root directory, a Stylesheet can be imported from a separate directory.

Directory

A Directory is the name for a store of files in within a File System. In an Operating System such as Windows, directories are more commonly known as Folders.

As a result of this, the terms Directory and Folder are often used interchangeably, regardless of the exact terminology that is used.

IDE

An Integrated Development Environment (IDE) is a piece of software that combines a text editor with tools specific to programming, such as linters, debuggers and compilers.

Alongside these tools, most IDEs also often support multiple programming languages with ease. In some cases, however, an IDE may only be designed with one programming language in mind.

Whilst code for any language can be written in pretty much any text editor (even Notepad!), text editors do not provide developers with the ability to properly format their code. In addition, a text editor such as Notepad does not include tools to assist in the debugging and/or linting of code, nor does it provide the ability to compile code into an executable program.

Some examples of IDEs include Visual Studio Code, Eclipse, JetBrains IntellJ IDEA and Code::Blocks.

Git

Git is an open-source version control system that is used for maintaining and keeping track of changes made to an application's source code.

Because Git is open-source, being licenced under version 2 of the GNU General Public License (GNU GPL), Git is free to use by anyone wishing to streamline how they manage a software development project.

When using Git, any files relating to a given project reside in a directory known as a repsitory. While repositories can reside on a local machine or a business's internal server, services such as GitHub and GitLab do exist to provide individuals or businesses with space to store their projects through the use of "cloud storage".

Below, I have included some examples of how you can use Git:

Creating a new repository

$ mkdir webdev-project
$ cd webdev-project
$ git init

Cloning an existing repository

$ mkdir webdev-project
$ cd webdev-project
$ git clone repository-url

Comitting files

$ git add index.html
$ git add styles/style.css
$ git commit

Pushing files to a repository

$ git push

Front End Developer

A Front End Developer is a developer that is primarily concerned with developing parts of an application or website that users will be interacting with. For example, anything that the user clicks on to invoke an action, types in to record data or reads to understand a given topic can all be considered to be examples of a Front End interface.

Whilst Front End Developers do produce interfaces that users can interact with, Front End Developers are not usually concerned with the overall design of a website. Instead, a Front End Developer typically only translates designs, which are usually made by a Web Designer, and turns them into an actual website using languages such as HTML, CSS and JavaScript.

Invensis

Back End Developer

A Back End Developer is a developer that is primarily concerned with developing the parts of an application, website or system that a user doesn't typically interact with. Essentially, unlike a Front End Developer, which translates designs produced for an application, website or system into working, user-facing code, a Back End Developer, produces the code that is responsible for carrying out the functions executed by a user, such as processing a transaction or booking a reservation at a restaurant.

In regards to Web Development, Back End Developers typically use languages such as SQL, PHP, Python and ASP.NET

Invensis

Full Stack Developer

A Full Stack Developer, as the name implies, is a developer that does the tasks that are typically carried out by both a Front End Developer and a Back End Developer respectively. As a result, this means that a Full Stack Developer is able to Front End Development using languages such as HTML, CSS and JavaScript, alongside doing Back End Development using languages such as SQL, PHP and Python.

Whilst a Full Stack Developer can choose to either work in either solely a Front End Development or Back End Development role, companies do often look for Full Stack Developers. More often than not, Full Stack Developers are usually paid more than a sole Front End Developer or Back End Developer.
